01 / Problem & Goal
01

Problem

Data customer, supplier, vendor, atau pihak eksternal belum punya tempat pengelolaan terpusat.

User berisiko mengetik ulang data pihak eksternal di berbagai proses bisnis.

Data kontak bisa jadi tidak konsisten, duplikatif, dan sulit ditelusuri historinya.

Kontak dibutuhkan sebagai fondasi untuk transaksi, hutang/piutang, accounting, dan reuse lintas aplikasi GroApp.

GgroApp access
01 / Problem & Goal
02

Goal

Menyediakan master data kontak yang terpusat di GroApp Access.

Membantu user melihat, menambah, membuka detail, dan memperbarui data kontak.

Menjaga data pihak eksternal lebih rapi, konsisten, dan mudah digunakan kembali.

Menyiapkan fondasi awal untuk kebutuhan accounting dan aplikasi GroApp lain.

04 / Aturan Bisnis
04

Aturan Bisnis

Kontak dikelola secara terpusat di GroApp Access sebagai shared master data.

Kontak berlaku dalam konteks satu perusahaan/tenant.

Kontak perusahaan A tidak bisa dilihat atau dikelola oleh perusahaan B.

Kontak dapat merepresentasikan individu maupun organisasi/usaha.

06 / Scope

Scope

✓In

  • Daftar Kontak
  • Detail Kontak
  • Tambah Kontak
  • Edit Kontak
  • Kesiapan fondasi reuse lintas aplikasi GroApp,

×Out

  • Pemakaian kontak langsung di transaksi penjualan, pembelian, biaya, hutang, atau piutang.
  • Pembangunan modul hutang/piutang.

↗Planned / Future

Umum (di groApp Access + Aplikasi Lainnya)

  • Hapus kontak
  • Import/Export kontak
  • Reuse kontak lintas aplikasi GroApp
  • Relasi kontak dengan user login / GroApp Access
  • Klasifikasi lanjutan kontak (selain tipe dan jenis)

Di GroApp Accounting

  • Data Statistik Keuangan kontak
  • Histori transaksi kontak
  • Pendataan legal/pajak kontak
  • Pendataan akun akuntansi/statistik kontak

07 / Daftar Fitur

Daftar Fitur

Nama Fitur Aktor Problem Goal
Daftar Kontak Pemilik Usaha, Administrator Perusahaan Data pihak eksternal belum punya titik akses utama yang terpusat. User dapat melihat dan mencari kontak yang sudah tercatat.
Detail Kontak Pemilik Usaha, Administrator Perusahaan Informasi kontak yang ringkas belum cukup untuk memahami konteks kontak. User dapat melihat informasi lengkap satu kontak sebelum digunakan atau diperbarui.
Tambah Kontak Pemilik Usaha, Administrator Perusahaan User berisiko mengetik ulang data pihak eksternal setiap kali dibutuhkan. User dapat membuat master data kontak yang terstruktur dan siap digunakan kembali.
Edit Data Kontak Pemilik Usaha, Administrator Perusahaan Data kontak bisa berubah dan menjadi tidak akurat jika tidak bisa diperbarui. User dapat menjaga data kontak tetap akurat, relevan, dan layak digunakan.
